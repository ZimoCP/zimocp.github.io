EXKMP

发表于 2019-08-23 | 分类于 OI

因为最近学各种字符串算法，觉得EXKMP不太好理解，担心以后忘了所以写一篇博客

EXKMP是什么、干什么的

KMP: D.E.Knuth，J.H.Morris和V.R.Pratt 3个人发明的一个算法
EX: Extended 拓展

用来计算一个字符串的任一后缀和另一个字符串的最长公共前缀

怎么做

首先先去学KMP（师兄的博客）

一些定义

KMP里面的p数组：一个字符串中区间[1,p[i]]和区间[i-p[i]+1,i]是完全一样的，也就是说以i为结尾的一段长为p[i]的字串和开头是一样的

EXKMP里面也有一个类似的数组，我们先管它叫ex (extend)，意义是串中[1,ex[i]]和[i,i+ex[i]+1]是完全一样的，也就是从开头拉出来ex[i]个和从i拉出来ex[i]个是完全一样的

首先确定：ex[1]=len不解释，所以我们直接从ex[2]开始

如果我们现在要求ex[i] (i>1)，我们先设一个k (1<=k<i)，意义是最大的可以使ex[k]已知的值（最后一个计算过的）
再定一个p=k+ex[k]-1，也就是k和前面匹配的字串的结尾，所以[1,p-k+1=ex[k]]和[k,p]是一样的

因为是完全一样的，所以我们从开头那边那个串也掏一段[i-k+1,p-k+1]和[i,p]匹配

接下来我们定一个l=ex[i-k+1]让[1,l]和[i-k+1,i-k+1+l-1 = i-k+l]匹配

接下来就不会有任何定义了

懒人结论

不想看过程的话结果就在这里：

  1. i+l < p+1的话ex[i]=l
  2. i+l > p+1的话ex[i]=p-i+1
  3. i+l = p+1的话只能暴力求，暴力的起点分两个情况
    • i>p 从i开始和开头暴力比较
    • i<p 从p开始，ex[i]=p-i+暴力的p后面的个数

开始证明

这个l还要分类讨论，实在是过于恶心

1: i-k+l < p-k+1

从图中可以明显的看出ex[i]=l
为什么绿点和黄点不能相等、不能是ex[i]=l+1呢？因为如果是这样的话l也就是ex[i-k+1]就会是在绿点那里而不是在l那里了

2: i-k+l > p-k+1

如图，ex[i]=p-i+1
为什么[1,p-i+1]就会匹配[i,p]呢？看图就能看出来紫色部分有着等量代换的作用。。。[i,p]=[i-k+1,p-k+1]=[1,p-i+1]
为什么蓝点和品红点不相等呢？因为如果相等的话p也就是ex[k]就会在品红点而不是p那里了

3: i-k+l = p-k+1

因为三个点不知道关系，所以只能暴力从p往后求
